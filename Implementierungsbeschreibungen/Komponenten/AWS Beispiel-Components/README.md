# README

## Einordnung

Die beiden vorliegenden AWS Component Definitions stellen einen ersten Versuch dar, produktspezifische OSCAL Components zu erstellen.

Ziel dieser Ausarbeitung ist es, exemplarisch aufzuzeigen, welche fachlichen und strukturellen Möglichkeiten das OSCAL Component Model grundsätzlich bieten kann. Die Dateien dienen daher nicht nur als inhaltliche Umsetzungshilfe, sondern auch als Demonstrator für den möglichen Einsatz produktspezifischer Components im Kontext von z.B. Grundschutz++.

## Zielsetzung

Mit den beiden Beispielen wurde bewusst versucht, die Möglichkeiten einer OSCAL Component möglichst weit auszuschöpfen. Dazu gehören insbesondere:

- die Abbildung produktspezifischer Umsetzungswege für abstrakte Anforderungen,
- die Verknüpfung zu referenzierten Anforderungen,
- die Nutzung technischer Beschreibungen, Hinweise und Quellenverweise,
- die Erprobung einer stärker anwendungs- und plattformspezifischen Strukturierung von Maßnahmen.

Die Dateien sollen damit vor allem verdeutlichen, was mit OSCAL Components theoretisch möglich ist und welche Formen der Wiederverwendung, Strukturierung und Nachvollziehbarkeit sich daraus ergeben können.

## Charakter der Ausarbeitung

Die Component Definitions sind als explorative Arbeitsstände zu verstehen. Sie verfolgen nicht den Anspruch, bereits ein abschließend konsolidiertes oder verbindliches Zielmodell darzustellen. Vielmehr sollen sie als Diskussions- und Entwicklungsgrundlage dienen.

Ein Teil der Inhalte wurde mit Unterstützung von KI erstellt oder überarbeitet. Trotz fachlicher Durchsicht und Nachbearbeitung wird daher ausdrücklich darauf hingewiesen, dass die Inhalte keinen Anspruch auf vollständige oder 100-prozentige fachliche Korrektheit erheben.

## Hinweise zur Verwendung

Vor einer produktiven, verbindlichen oder institutionellen Weiterverwendung sollten die Inhalte daher in jedem Fall fachlich geprüft, validiert und bei Bedarf überarbeitet werden. Dies gilt insbesondere für:

- die fachliche Passgenauigkeit der referenzierten Anforderungen,
- die technische Umsetzbarkeit einzelner Beschreibungen,
- die Vollständigkeit und Aktualität der Quellenverweise,
- die formale und inhaltliche Eignung für den vorgesehenen Einsatzzweck.

## Fazit

Die beiden AWS Component Definitions sind in erster Linie als Demonstration und Erprobung zu verstehen. Sie sollen greifbar machen, wie produktspezifische OSCAL Components aufgebaut sein können und welchen Mehrwert ein solcher Ansatz perspektivisch bieten könnte.
